Table

Types

TableConfig <T>

Underlined attributes are required.
typescript 
export interface TableConfig<T> {
	id: string;
	data: Writable<T[]>;
	toggle?: boolean;
	fitToScreen?: boolean;
	height?: null | number;
	columns?: Columns;
	pageSizes?: number[];
	defaultPageSize?: number;
	optionsComponent?: typeof SvelteComponent;
}
id:
string

Unique identifier for the table to generate unique IDs for the filters.

data:
Writable <T[]>

A writable store of the type T[]. Any changes in the store will be reflected in the table.

toggle:
boolean

Whether the fitToScreen toggle should be visible. false by default.

resizable:
"rows", "columns" or "both"

Whether rows, columns or both should be resizable. Not resizable by default.

fitToScreen:
boolean

Whether the table takes the whole width. true by default.

height:
number

Custom height for the table in pixels.

optionsComponent:
SvelteComponent

Custom Svelte component to apply actions on a specific row. Table will not have an options column if no optionsComponent was provided.

pageSizes:
number[]

An array of page sizes to be used for the table. By default, page sizes are 5, 10, 15, 20.

defaultPageSize:
number

Default page size to be used for the table upon rendering. By default, default page size is 10.

columns:
Columns

An object with configuration for specific columns. Columns object is described below.

Columns

typescript 
export interface Columns {
	[key: string]: Column;
}
[key: string]:
Column

Each key of this object is an accessor in the data object. If a certain key is not present, defaults will be applied for that accessor/column. If a key is present, defaults will be replaced with provided data.

Column

typescript 
export interface Column {
	header?: string;
	exclude?: boolean; // false by default
	instructions?: ColumnInstructions;
	disableFiltering?: boolean; // false by default
	disableSorting?: boolean; // false by default
	colFilterFn?: ColumnFilterFn;
	colFilterComponent?: typeof SvelteComponent;
}
header:
string

Custom header to be displayed for the column. If not provided, header will be the same as the accessor.

minWidth:
number

Minimum width in pixels for the column. By default, the column will be as wide as the content.

fixedWidth:
number

Fixed width in pixels for the column. By default, the column will be as wide as the content. Setting this will override the resizable property for the column.

exclude:
boolean

Whether to exclude the column from the table or not. By default, columns are not excluded. An excluded column will still be available in the data object to apply actions.

disableFiltering:
boolean

Whether to disable filtering for the column or not. By default, filtering is enabled.

disableSorting:
boolean

Whether to disable sorting for the column or not. By default, sorting is enabled.

colFitlerFn:
ColumnFilterFn

Filtering function to be applied on the column. By default, basic number/string filter is applied.

colFilterComponent:
SvelteComponent

Custom Svelte component to be used for filtering the column. By default, basic TableFilter component is rendered for the filters.

instructions:
ColumnInstructions

Configuration for the column's appearance, sorting and filtering functionalities.

ColumnInstructions

typescript 
export interface ColumnInstructions {
	toStringFn?: (value: any) => string;
	toSortableValueFn?: (value: any) => string | number;
	toFilterableValueFn?: (value: any) => string | number | Date;
}
toStringFn:
(value: any) => string

Custom toString function to be applied on the column.

toSortableValueFn:
(value: any) => string | number

Custom function to return either a string or a number value to have a sortable column.

toFilterableValueFn:
(value: any) => string | number | Date

Custom function to return either a string, number or a Date value to have a filterable column. Filter options are based on the returned value.

renderComponent:
SvelteComponent

renderComponent overrides toStringFn.

Every renderComponent will get 2 props:

row:
Object

The current row in the table. In other words, a single object from the data store.

value:
any

The value of the current cell.

Action Dispatchers

on:action

(action: CustomEvent<any>) => {}

To dispatch actions and listen to dispatchers you need to create a table with your own optionsComponent component.

Every optionsComponent will get 2 props:

row:
Object

The current row in the table. In other words, a single object from the data store.

dispatchFn:
(obj: any) => dispatch('action', obj)

A function that takes an object and dispatches an action with the name action. The passed object is accessible by the parent component through the action.detail property.

				fixedWidth: 120
			}

		pageSizes: [1, 3, 5],
		defaultPageSize: 5
	};
</script>

	config={usersNoRolesConfig}

<script lang="ts">
	import { Table } from '@bexis2/bexis2-core-ui';
	import type { TableConfig } from '@bexis2/bexis2-core-ui';

	import { usersMissingIDsStore } from './data';
	import type { UserMissingID } from './data';

	const missingValues: { [key: number]: string } = {
		32655: 'NA',
		32654: 'NM',
		32653: 'ND'
	};

	const usersMissingIDsConfig: TableConfig<UserMissingID> = {
		id: 'usersMissingIDs',
		data: usersMissingIDsStore,
		columns: {
			id: {
				header: 'ID',
				instructions: {
					toStringFn: (key: number) => {
						if (key in missingValues) {
							return missingValues[key];
						}
						return key.toString();
					}
				}
			}
		}
	};
</script>

<Table config={usersMissingIDsConfig} />

		fitToScreen: false,
